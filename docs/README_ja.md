## ✨ 特徴

- 🤖 **マルチエージェントパイプライン** — 9つの専門エージェントが担当：企画、世界観構築、プロット作成、執筆、レビュー、推敲、総括
- 📋 **マルチドラフトアウトライン比較** — 複数のスタイル（ドラマチック型/文学型/商業型）でアウトラインを生成し、選択または統合
- 🔍 **並列品質レビュー** — 3つの並列レビュアー（一貫性/文体/伏線）が各章を分析、信頼度スコア付き
- ✨ **自動推敲ループ** — 章を自動評価し設定可能な品質閾値に達するまで反復改善
- 📍 **巻間チェックポイント** — 巻の間で一時停止し、進捗を確認して方向を調整
- 📊 **完結サマリーレポート** — 小説完成後に完全なレポートを自動生成：プロット分析、キャラクターアーク、統計データ、続編の伏線
- 🌐 **11以上の言語を内蔵** — 英語、簡体/繁体中国語、日本語、韓国語、ベトナム語、タイ語、漢文、古典日本語、ラテン語、サンスクリット語
- ➕ **自己拡張型 i18n** — 実行時に新しい言語を追加可能！
- 🔌 **OpenAI 互換バックエンド** — OpenAI 互換 API をサポート
- ⚡ **並列実行** — 世界観設定、スタイルガイド、アウトラインドラフト、品質レビューをすべて並列実行
- 💾 **チェックポイント再開** — 完了済みの章を自動検出し、中断箇所から再開
- 📝 **豊富なメタデータ追跡** — 進捗、章要約、伏線トラッカー、キャラクター状態をMarkdownで管理
- 🎨 **カスタマイズ可能なパラメータ** — すべてのハイパーパラメータを config に集約；ユーザー個別の設定（APIキー含む）は `user_config.json` に分離（git-ignore済み）

### エージェント一覧

| エージェント | 役割 |
|-------------|------|
| **PlannerAgent** | マルチターン対話で小説の要件を収集（ジャンル、テーマ、構成、キャラクターなど） |
| **WorldbuildingAgent** | 世界観設定、キャラクタープロフィール、場所、タイムライン、能力/技術体系を生成（並列） |
| **StyleGuideAgent** | 小説プランに基づく執筆スタイルガイドを作成 |
| **OutlineAgent** | マスターアウトライン（単一またはマルチドラフト比較）と巻別アウトラインを生成 |
| **WriterAgent** | アウトライン、スタイルガイド、連続性コンテキストに従って章を執筆 |
| **PostWriteAgent** | 完成した章を分析し、要約を生成、進捗/伏線メタデータを更新 |
| **QualityReviewerAgent** | 3つの並列レビュアー（一貫性/文体/伏線）、信頼度スコア付き |
| **PolishAgent** | 自己反復推敲：品質評価 → 改善 → 再評価、閾値達成まで |
| **FinalSummaryAgent** | 小説完成後の完全レポート：統計、キャラクターアーク分析、続編の伏線 |

## 🚀 クイックスタート

### 1. 依存パッケージのインストール

```bash
pip install -r requirements.txt
```

### 2. 設定ファイルの準備

```bash
cp user_config.example.json user_config.json
```

`user_config.json` を編集し、API 認証情報やお好みの設定を記入してください（上書きしたい項目のみ記載すれば OK です）。

> ⚠️ `user_config.json` は git-ignore されています。API キーをコミットしないでください！
> 利用可能な全項目は `user_config.example.json` を参照してください。

#### 作品別設定（オプション）

各小説プロジェクトに独自の `novel_config.json` を配置して、執筆関連パラメータを上書きできます：

```bash
cp novel_config.example.json output/あなたの小説名/novel_config.json
```

> **設定読み込み順序**：`config.py` デフォルト → `user_config.json`（グローバル） → `novel_config.json`（作品別）
>
> セキュリティのため、`novel_config.json` 内の API 関連設定は **自動的にスキップ** されます。API 設定はグローバルの `user_config.json` で行ってください。
> 利用可能な作品別項目は `novel_config.example.json` を参照してください。

### 3. 実行

```bash
python main.py
```

以下の手順で案内されます：

1. **言語選択** — 小説の言語を選択（新しい言語の追加も可能！）
2. **プロジェクト設定** — 小説プロジェクトに名前をつける
3. **操作選択** — フルパイプライン、再開、または個別フェーズの実行

### 操作モード

| モード | 説明 |
|--------|------|
| **フルパイプライン** | 企画から最終章までの完全なフロー |
| **再開** | 最後のチェックポイントから続行 |
| **企画のみ** | インタラクティブ対話で小説を設計 |
| **世界観構築のみ** | 世界観設定とスタイルガイドを生成 |
| **アウトラインのみ** | マスター/巻別アウトラインを生成 |
| **執筆のみ** | 指定範囲の章を執筆 |

## 🌐 新しい言語の追加

言語メニューで `➕ 新しい言語を追加` を選択し、言語情報を入力するだけ。LLM がすべての UI 文字列、プロンプト、テンプレートを自動翻訳し、新しいロケールファイルを即座に生成します。

`locales/` ディレクトリに手動で `.py` ファイルを配置することもできます — 次回起動時に自動的に検出されます。

## ⚙️ 設定パラメータ

### コアパラメータ

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| `TEMPERATURE` | 0.7 | LLM サンプリング温度 |
| `TOP_P` | 0.6 | 核サンプリング閾値 |
| `MAX_OUTPUT_TOKENS` | 32768 | LLM 応答あたりの最大トークン数 |
| `CHAPTER_MIN_WORDS` | 3000 | 章あたりの最小文字数 |
| `CHAPTER_MAX_WORDS` | 4000 | 章あたりの最大文字数 |
| `MAX_PARALLEL_WORKERS` | 3 | 並列タスク数 |
| `MAX_RETRIES` | 5 | API 呼び出しリトライ回数 |

### 高度な機能

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| `OUTLINE_DRAFT_COUNT` | 3 | アウトラインドラフト数（1で無効化） |
| `ENABLE_PARALLEL_REVIEW` | True | 各章の後に3つの並列品質レビューを実行 |
| `ENABLE_POLISH_LOOP` | True | 評価→改善ループで章を自動推敲 |
| `POLISH_QUALITY_THRESHOLD` | 8 | 推敲ループ終了に必要な品質スコア (1-10) |
| `ENABLE_VOLUME_CHECKPOINT` | True | 巻の間で一時停止してユーザー確認 |
| `ENABLE_FINAL_SUMMARY` | True | 小説完成後に完全レポートを生成 |
